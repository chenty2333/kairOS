# Kairos

A modern, multi-architecture operating system kernel written in C.
This kernel is an experiment in AI-Native OS development. 100% of the code was generated by AI agents (Claude Code, Gemini, Codex) under human architectural guidance.

## Goals

- **Multi-architecture**: RISC-V 64, AArch64, x86_64
- **Real hardware**: USB keyboard, NVMe storage, framebuffer display
- **POSIX-like**: Run busybox, TCC, and eventually GCC
- **Clean design**: Modular, well-documented, maintainable

## Architecture Support

| Architecture | Status | Bootloader |
|-------------|--------|------------|
| RISC-V 64   | Primary development | Limine |
| AArch64     | Planned | Limine |
| x86_64      | Planned | Limine |

## Building

```bash
make ARCH=riscv64    # Build for RISC-V
make ARCH=x86_64     # Build for x86_64
make run             # Run in QEMU
```

## Directory Structure

```
kairos/
├── build/          # Build outputs (per-arch)
├── busybox/        # Busybox sources
├── docs/           # Design documents
├── kernel/         # Kernel source
│   ├── arch/       # Architecture-specific code
│   ├── boot/       # Bootloader integration (Limine)
│   ├── core/       # Core subsystems (sched, mm, proc, syscall, time)
│   ├── drivers/    # Device drivers (block/net/usb/virtio/...)
│   ├── fs/         # VFS and filesystems (ext2/fat32/initramfs/procfs)
│   ├── ipc/        # IPC subsystems
│   ├── lib/        # Kernel libraries (rbtree, string, etc.)
│   └── include/    # Kernel headers
├── scripts/        # Build scripts and helpers
├── third_party/    # External dependencies
├── tools/          # Build tools and utilities
├── user/           # User space
│   ├── build/      # Userland build outputs
│   ├── init/       # Init process
│   ├── initramfs/  # Initramfs contents
│   ├── libc/       # C library port (musl)
│   └── shell/      # Basic shell
├── disk.img        # Disk image (generated)
├── limine.cfg      # Bootloader configuration
└── qemu-virt.dtb   # QEMU DTB (riscv64)
```
